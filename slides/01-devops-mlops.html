The Gap

The notebook is not a product

"It works on my machine" — the most dangerous phrase in ML.
A model trained in a notebook can't serve real users — it needs an API, a container, a pipeline.
Without versioning, you can't reproduce your best result — or roll back from your worst.
Models degrade silently as real-world data drifts — no monitoring means no warning.
This workshop gives you the bridge: DevOps & MLOps practices that turn your project into a product.
Module 01

What is DevOps?

A culture and set of practices that unifies development, operations, and delivery.

People

Shared ownership

Dev and Ops work together, not in silos. The team that builds it is responsible for running it.

Process

Continuous delivery

Small, frequent changes over big, rare releases. Every commit flows through an automated pipeline.

Tools

Automation everywhere

Git, Docker, GitHub Actions, Terraform — the stack that removes manual toil from code to production.

MLOps maturity levels

Where does your graduation project sit today?

0

Manual everything

Notebook only. No versioning, no API, no reproducibility. Your laptop IS the server.

2

Automated pipeline

CI/CD triggers training + testing + container build on every push. Models are versioned.

1

Scripted + containerized

Training is a script. Model served via Docker. Deployment is still manual. ← Target for today

3

Full MLOps

Automated retraining on data drift. Model registry. A/B testing. Self-healing production.
