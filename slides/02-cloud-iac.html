Module 02

The three cloud primitives

Everything in AWS (and any cloud) is one of three things: compute, storage, or network.

Compute · EC2

Virtual machines on demand

Rent a CPU (or GPU) by the hour. Your ML training job and your API server both run here.

t3.medium · g4dn.xlarge
Storage · S3

Object storage for everything

Unlimited files at pennies per GB. Model artifacts, training datasets, Docker layer cache — all here.

s3://my-bucket/models/v1/
Network · VPC + ALB

Your private network + load balancer

VPC isolates your resources. ALB (Application Load Balancer) routes user traffic to your containers.

api.myproject.com → :8000

Infrastructure as Code

Stop clicking. Start committing.

Define your cloud infrastructure in files, version it in Git, and let automation do the rest. Terraform is the most widely used IaC tool.

Version control

Infrastructure lives in Git alongside your code. Every change is auditable, reviewable, and reversible with a single command.

Reproducibility

The same Terraform config creates identical environments in dev, staging, and production. No more "it works in staging but fails in prod."

Speed

Spinning up a GPU instance for training goes from a 10-minute console session to one command: terraform apply

terraform / main.tf

Declare an EC2 instance in code

# main.tf — provision an EC2 instance for ML training

provider "aws" {
  region = "us-east-1"
}

resource "aws_instance" "ml_trainer" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t3.medium"    # swap to g4dn.xlarge for GPU

  tags = {
    Name = "ml-training-server"
    Env  = "dev"
  }
}
