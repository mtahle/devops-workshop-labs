The problem

Why model versioning matters

Different models perform differently on different data slices — you need to compare runs side by side.
A deployed model can degrade silently as real-world data drifts from the training set.
Without versioning, "rolling back" means re-training from scratch — you've lost the previous artifact.
Teams need answers: which model is live? trained on what data? by whom? when? with what accuracy?
MLflow answers all of these: experiment tracking + model registry = reproducibility + accountability.

The three environments

Code flows one direction: Dev → Staging → Production. No shortcuts.

Development

Where you experiment

Local Docker, fast iteration, broken code is acceptable. This is your sandbox.

localhost:8000
Staging

Where you validate

Same config as production. CI must pass. Stakeholders review here before release.

staging.myproject.com
Production

Where users live

Auto-deploy from main only. Monitor always. Rollback must be instant and practiced.

api.myproject.com
